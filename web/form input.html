오늘은 FORM TAG에 대해 배워보겠습니다!


<form>이란 사용자의 입력을 위한 HTML폼을 정의할때 사용되어 집니다.

좀 더 자세히 들어가 볼까요?
  • 먼저 우리가 만들 Html을 위해 FORM TAG의 종류에 대해서 알아둬야 겠죠?
  • accept-charset,action,enctype,method,name,target과 같은 종류가 있다.
  • 태그의 기본구조는 예시로

    • <form>

    입력요소

    </form>

    이렇게 기본틀을 만들수잇다.

    수업시간에 우리는 method와action을 배웠는데 오늘 이시간에는 이것들 위주로 할거예요!

  • <form method="getIpost">

    • method속성은 from데이터 전송시에 http메소드를 지정하여 주고, 데이터 전송시 사용되는 http메소는 post와 get이 있습니다.

  • get방식

    • 1.이름과 값을 가지는 한쌍의 형식으로 url에 데이터를 추가

    2.url길이는 약 3,000자로 제한되어진다

    3.url에서 데이터가 바로 보여짐

  • post방식

    • 1.데이터 길에에 제한이 없음

    2.url에서 데이터가 보이지않음

    get방식을 사용할 때에는 보안이 중요 하지 않거나 노출되어 상관 없는 데이터를 보낼때에 사용되어지고,

    post방식은 중요한 데이터를 보낼때나 노출되어져서는 안되는 데이터를 전송시 사용합니다.

    보안상으로 보자면 post방식을 이용하여 코딩하는 것이 좋지만 개발 상황에 따라 get방식도 많이 사용합니다.

    form태그에서 method방식을 지정하지 않을경우 기본값으로 get방식이 지정되니 이점을 주의 해야 해요.


    다음은 FORM TAG - ACTION 태그에 대해서 배워볼거예요!
  • <form action="url">

    • action속성은 from데이터들이 전송되어지면 어느 페이지에 결과를 뿌려줄지 결정해주는 역할을 해준답니다!

  • a.html 페이지
  • b.html 페이지

    • 이 두 웹 페이지가 있으면 url에 각각 다른이름을 써넣음으로써 각각 다른 페이지에 다른 결과를 부여하는 것이죠

  • a.html의 경우

    • <form action="a.html">

    <input type="text" name="soso" value="todos"></form>

  • b.html의 경우

    • <form action="b.html">

    <input type="text" name="soso" value="todos"></form>

    이처럼 type이 text인 soso라는 이름을 가진 input태그는 from 전송시 "todos"라는 값을 보냅니다.

    하지만 각각 action 값을 다르게 지정했죠?!

    첫번째 form에서 전송되어진 "todos"라는 값은 action 속성에서 지정한데로 a.html페이지로 전송되어지고

    두번째 form에서 전송되어진 "todos"라는 값은 b.html페이지로 전송되어집니다~

    이상 form 태그 설명이였습니다~
