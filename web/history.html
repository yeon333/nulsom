html역사?


html은 발전하는 언어이다.

인터넷 서비스의 하나인 월드 와이드 웹을 통해 볼 수 있는 문서를 만들 때 사용하는 프로그래밍 언어의 한 종류이다.

특히 하이퍼텍스트를 작성하기 위해 개발되었으며, 인터넷에서 웹을 통해 접근되는 대부분의 웹 페이지들은 HTML로 작성된다.


HTML 1.0 과 2.0


HTML은 웹의 아버지라 불리는 팀 버너스 리에 의해 설계되어 1990년에 개발되었다.

1993년 8월에 표준화 되었으며, 이후 HTML 2.0까지 IETF에 의해 관리되었다.


*IETF이란?*


국제 인터넷 표준화 기구(Internet Engineering Task Force)를 말한다.

HTML 3.2 ~ 4.0

HTML 3.2가 1997년 1월에 발표되엇으며 이때부터 W3C 주도하에 표준이 제정되어 관리되었다.

같은해 12월 W3C는 HTML 4.0 을 발표하면서 더 이상 HTML 업그레이드를 하지 않겠다고 선언하고

HTML WG를 중단시켰는데, 이를 HTML로 만들어진 웹 페이지를 차세대 웹 표준으로 추진하고 있는
<
XML로 대체하기 위하여 내린 결단이였다.


*W3C란?*


(world wide web consortium)

W3C는 HTML표준안을 확정하는 곳으로 인터넷 웹사이트의 향방을 손에 쥐고 있으며,

회원기구와 직원들 그리고 공공기관이 협력하여 웹 표준을 개발하고 있다.

IBM, 마이크로소프트, 넷스케이프, 소니 등 내로라하는 일반 기업을 비롯해

비영리기관, 산업기관,정부기구 등이 W3C의 회원사 목록을 구성한다.

HTML 4.01과 XML 1.1

W3C는 HTML WG를 중단 시킨 후 약 2개월 뒤인 1998년 2월에 XML 1.0을 발표하고

같은 해 7월에는 XML의 표준화를 목표로 두 번째 HTML WG를 설립

1999년 12월에 HTML의 마지막 버전인 4.01을 발표하고 발고 다음달에

HTML 4.01태그를 그대로 사용하고 문법만 XML을 따르도록 만든 XHRML 1.0을 발표한후 2001년에는 XHTML 1.1 발표

HTML 4.01과 XHTML 1.1 그후

HTML 4.01과 XHTML 1.1 이 발표된 후 HTML 표준은 눈에 띄는 발전이 없어 몇 년이 지나갔고

여전히 HTML이나, MIME 타입을 HTML로 선언한 XHTML이나 W3C가 추구하는 목적과는 다른 방향으로 웹페이지가 계속 만들어짐



W3C의 HTML5표준호

W3C는 2006년 7월에 XHTML의 다음버전인 XHTML 2.0의 초안을 발표한 등 WHAT WG가 설립된 이후에도 몇 년 동안은

원래의 계획대로 XHTML을 계속 발전시켜 나가려 노력하였으나, 결국 이상(XHTML)과 현실(HTML) 사이의 장벽에 부딪쳐

2006년 10월 W3C의 설립자는 '팀 버너스 리'는 W3C와 WHAT WG가 공동으로 HTML5 제정에 함께하자는 발표를 함
